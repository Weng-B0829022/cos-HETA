================================================================================
  1. 圖資料結構
================================================================================

Python 版本使用 NetworkX，底層是 dict-of-dict：

  G._adj = {
      0: {1: {}, 2: {}, 3: {}},
      1: {0: {}, 2: {}},
      ...
  }

  - 每個節點是一個 Python dict 物件
  - 每條邊是一個空 dict {}（可存邊屬性）
  - 存取鄰居：G._adj[u]           → 一次 hash 查找
  - 存取邊：  G._adj[u][v]        → 兩次 hash 查找
  - 記憶體：每條邊約 200~400 bytes（Python 物件頭 + hash table）
  - 遍歷鄰居：iterator，每個 yield 都有 Python 函數呼叫開銷

  ┌────────────────┬──────────────────┬──────────────────┬──────────────────┐
  │ 操作           │ 資料結構         │ 時間複雜度       │ 空間複雜度       │
  ├────────────────┼──────────────────┼──────────────────┼──────────────────┤
  │ 建圖           │ dict-of-dict     │ O(n+m)           │ ~400 bytes/邊    │
  │ 存取鄰居       │ Python dict      │ O(deg) + GIL     │ —                │
  │ 記憶體佔用     │ Python 物件      │ —                │ ~400 bytes/邊    │
  └────────────────┴──────────────────┴──────────────────┴──────────────────┘

--------------------------------------------------------------------------------

C 版本使用 CSR（Compressed Sparse Row）：

  offset   = [0, 3, 5, 7, ...]       大小 n+1
  adj_list = [1,2,3, 0,2, 0,1, ...]  大小 2m，所有鄰居連續存放

  節點 u 的鄰居 = adj_list[ offset[u] .. offset[u+1]-1 ]

  - 所有鄰居連續存放在同一塊記憶體
  - 遍歷節點 u 的鄰居 = 讀取連續 int 陣列，CPU prefetch 自動載入
  - 記憶體：每條邊 8 bytes（兩個 int），比 Python 少 50 倍
  - 無 GIL、無 hash、無 Python 物件開銷

  ┌────────────────┬──────────────────┬──────────────────┬──────────────────┐
  │ 操作           │ 資料結構         │ 時間複雜度       │ 空間複雜度       │
  ├────────────────┼──────────────────┼──────────────────┼──────────────────┤
  │ 建圖           │ int[] 陣列 x3    │ O(n+m)           │ ~8 bytes/邊      │
  │ 存取鄰居       │ 連續 int 陣列    │ O(deg)，高 cache │ —                │
  │ 記憶體佔用     │ 原生 int         │ —                │ ~8 bytes/邊      │
  └────────────────┴──────────────────┴──────────────────┴──────────────────┘

  改進倍數：存取速度 5~10x，記憶體 ~50x


================================================================================
  2. 圖複製
================================================================================

switching randomization 每次執行前需要完整複製一份圖。
1000 個隨機網路 = 1000 次複製。

Python：
  Gr = G.copy()
  - 底層呼叫 copy.deepcopy，遞迴複製每一層 dict
  - 每個 Python 物件都要重新 malloc + 初始化引用計數
  - n=34, m=78 的 karate club：約建立 34+78 個 dict
  - 重複 1000 次 = 1000 次 dict deepcopy，觸發 GC 壓力

  ┌────────────────────┬──────────────────────────────────────────────────────┐
  │ 資料結構           │ Python dict deepcopy                                 │
  │ 時間複雜度         │ O(n+m)，常數極大（Python 物件 malloc）               │
  │ GC 壓力            │ 高（每次產生大量短命物件）                           │
  └────────────────────┴──────────────────────────────────────────────────────┘

C：
  Graph *graph_copy(const Graph *g) {
      memcpy(c->offset,   g->offset,   (n+1) * sizeof(int));
      memcpy(c->adj_list, g->adj_list,  2*m  * sizeof(int));
      memcpy(c->degree,   g->degree,    n    * sizeof(int));
  }
  - 三次 memcpy，複製連續記憶體區塊
  - 作業系統層級批次記憶體複製，速度接近記憶體頻寬上限
  - n=34, m=78：複製 (35+156+34) x 4 = 900 bytes，一次 cache line 完成

  ┌────────────────────┬──────────────────────────────────────────────────────┐
  │ 資料結構           │ memcpy 連續 int 陣列                                 │
  │ 時間複雜度         │ O(n+m)，常數 = 4 bytes/元素                          │
  │ GC 壓力            │ 無（只有 3 次 malloc）                               │
  └────────────────────┴──────────────────────────────────────────────────────┘

  改進倍數：~100x


================================================================================
  3. has_edge 邊存在查詢
================================================================================

switching randomization 每步需查詢 (a,d) 和 (c,b) 是否已存在。
總呼叫次數 = Q x m x n_random = 100 x 78 x 1000 = 7,800,000 次

Python：
  Gr.has_edge(a, d)
  # 底層：return d in G._adj[a]
  - 兩層 Python dict 查找
  - 每次：Python 函數呼叫 + hash(a) + dict lookup + hash(d) + dict lookup
  - 單次耗時約 100~200 ns

  ┌────────────────────┬──────────────────────────────────────────────────────┐
  │ 資料結構           │ Python dict-of-dict                                  │
  │ 時間複雜度         │ O(1) 平均，但常數大                                  │
  │ 單次耗時（估算）   │ ~150 ns                                              │
  └────────────────────┴──────────────────────────────────────────────────────┘

C：
  // adj_bs[u] 是節點 u 的鄰居 bitset
  static inline int bs_test(const Bitset *bs, int node) {
      return (bs->words[node / 64] >> (node % 64)) & 1;
  }
  if (bs_test(adj_bs[a], d)) continue;
  - 一次陣列存取 + 一次位元移位 + 一次 AND
  - 無函數呼叫（inline）、無 hash、無 Python 物件
  - 單次耗時約 1~3 ns

  ┌────────────────────┬──────────────────────────────────────────────────────┐
  │ 資料結構           │ Bitset（uint64_t 陣列），每個節點一個                │
  │ 時間複雜度         │ O(1)，常數極小                                       │
  │ 單次耗時（估算）   │ ~2 ns                                                │
  └────────────────────┴──────────────────────────────────────────────────────┘

  改進倍數：~75x


================================================================================
  4. BFS / k 層鄰居計算
================================================================================

每條邊每層需要 2~4 次 BFS，是 CNR 計算的核心子程序。

Python：
  def get_kth_layer_neighbors(G, node, exclude, k):
      visited  = {node, exclude}   # Python set，動態分配
      frontier = {node}            # Python set

      for _ in range(1, k):
          next_frontier = set()    # 每層都 new 一個 set
          for n in frontier:
              for nb in G.neighbors(n):      # iterator，有呼叫開銷
                  if nb not in visited:      # hash 查找
                      next_frontier.add(nb)  # hash 插入
                      visited.add(nb)
          frontier = next_frontier

      kth = set()                  # 再建一個 set
      return kth

  ┌─────────────────────┬─────────────────────────────────────────────────────┐
  │ visited 結構        │ Python set（hash table）                            │
  │ queue 結構          │ Python set frontier（每層新建）                     │
  │ 鄰居遍歷            │ Python iterator                                     │
  │ 時間複雜度          │ O(n+m)，常數大                                      │
  │ 臨時物件            │ 每次呼叫建立 3~5 個 set，全部進 GC                 │
  └─────────────────────┴─────────────────────────────────────────────────────┘

C：
  void compute_kth_layer(const Graph *g, int u, int exclude, int k,
                         Bitset *out, BFSBuffer *buf) {
      // buf 是預分配的 thread-local 緩衝區，不需要 malloc
      memset(buf->visited,   0,  n * sizeof(int));
      memset(buf->layer_buf, -1, n * sizeof(int));

      buf->visited[u]       = 1;
      buf->visited[exclude] = 1;
      buf->queue[tail++]    = u;

      while (head < tail) {
          int cur = buf->queue[head++];
          for (int i = g->offset[cur]; i < g->offset[cur+1]; i++) {
              int nb = g->adj_list[i];       // 連續記憶體存取
              if (!buf->visited[nb]) {       // 直接陣列索引，O(1)
                  buf->visited[nb]   = 1;
                  buf->layer_buf[nb] = cur_layer + 1;
                  buf->queue[tail++] = nb;
                  if (cur_layer + 1 == k) bs_set(out, nb);
              }
          }
      }
  }

  ┌─────────────────────┬─────────────────────────────────────────────────────┐
  │ visited 結構        │ int[] 陣列，直接索引                                │
  │ queue 結構          │ int[] 靜態陣列，head/tail 指標                      │
  │ 鄰居遍歷            │ CSR 連續 int 陣列                                   │
  │ 時間複雜度          │ O(n+m)，常數極小                                    │
  │ 臨時物件            │ 0（全部預分配，memset 重置）                        │
  └─────────────────────┴─────────────────────────────────────────────────────┘

  改進倍數：~10~20x（BFS 本身）；臨時記憶體分配從 O(n) 降為 O(1)


================================================================================
  5. 集合交集（共同鄰居計算）
================================================================================

CNR 計算的核心運算，每條邊每層執行 1~3 次集合交集。

Python：
  V1_u = set(G.neighbors(u)) - {v}   # 建立 set，再做差集
  V1_v = set(G.neighbors(v)) - {u}
  inter = len(V1_u & V1_v)           # 掃描較小 set，逐元素 hash 查找
  denom = min(len(V1_u), len(V1_v))

  ┌─────────────────────┬─────────────────────────────────────────────────────┐
  │ 資料結構            │ Python set（hash table）                            │
  │ 交集演算法          │ 掃描較小集合，逐元素 hash 查找                      │
  │ 時間複雜度          │ O(min(deg_u, deg_v))                                │
  │ 臨時物件            │ 每次呼叫建立 2~3 個 set                             │
  └─────────────────────┴─────────────────────────────────────────────────────┘

C：
  int bs_intersect_count(const Bitset *a, const Bitset *b) {
      int count = 0;
      for (int i = 0; i < n_words; i++) {
          count += __builtin_popcountll(a->words[i] & b->words[i]);
      }
      return count;
  }

  // AVX2 版本：一次處理 256 bits（4 個 uint64_t）
  __m256i va = _mm256_loadu_si256((__m256i *)(a->words + i));
  __m256i vb = _mm256_loadu_si256((__m256i *)(b->words + i));
  __m256i vc = _mm256_and_si256(va, vb);   // 256-bit AND

  ┌─────────────────────┬─────────────────────────────────────────────────────┐
  │ 資料結構            │ Bitset（uint64_t[] 連續陣列）                       │
  │ 交集演算法          │ bitwise AND + POPCNT                                │
  │ 時間複雜度          │ O(n/64)，與節點度數無關                             │
  │ AVX2 路徑           │ O(n/256)                                            │
  │ 臨時物件            │ 0                                                   │
  └─────────────────────┴─────────────────────────────────────────────────────┘

  改進倍數：~64x（一般）/ ~256x（AVX2）


================================================================================
  6. 共同鄰居比例 CNR 計算
================================================================================

對應論文公式 (1)(2)，整個 HETA 中呼叫次數最多的函式。

Python：
  def compute_common_neighbor_ratio(G, u, v, k):
      if k == 1:
          V1_u = set(G.neighbors(u)) - {v}
          V1_v = set(G.neighbors(v)) - {u}
          return len(V1_u & V1_v) / min(len(V1_u), len(V1_v))

      Vk_u  = get_kth_layer_neighbors(G, u, v, k)     # BFS → set
      Vk_v  = get_kth_layer_neighbors(G, v, u, k)     # BFS → set
      Vk1_u = get_kth_layer_neighbors(G, u, v, k-1)   # BFS → set（重算！）
      Vk1_v = get_kth_layer_neighbors(G, v, u, k-1)   # BFS → set（重算！）

  ┌──────────────────────┬──────────────┬──────────────────────────────────────┐
  │ 指標                 │ k=1          │ k>1                                  │
  ├──────────────────────┼──────────────┼──────────────────────────────────────┤
  │ BFS 次數             │ 2            │ 4（Vk-1 無快取，重算）               │
  │ 臨時 set 數          │ 2~3          │ 4~6                                  │
  │ 快取                 │ 無           │ 無                                   │
  └──────────────────────┴──────────────┴──────────────────────────────────────┘

C：
  double compute_cnr(const Graph *g, int u, int v, int k,
                     BFSBuffer *buf,
                     Bitset *bs_k_u,  Bitset *bs_k_v,
                     Bitset *bs_k1_u, Bitset *bs_k1_v) {
      // 所有 Bitset workspace 由呼叫者預分配，反覆重用
      compute_kth_layer(g, u, v, k,   bs_k_u,  buf);
      compute_kth_layer(g, v, u, k,   bs_k_v,  buf);
      compute_kth_layer(g, u, v, k-1, bs_k1_u, buf);
      compute_kth_layer(g, v, u, k-1, bs_k1_v, buf);

      int inter_kk  = bs_intersect_count(bs_k_u,  bs_k_v);
      int inter_k1k = bs_intersect_count(bs_k1_u, bs_k_v);
      int inter_kk1 = bs_intersect_count(bs_k_u,  bs_k1_v);
  }

  ┌──────────────────────┬──────────────┬──────────────────────────────────────┐
  │ 指標                 │ k=1          │ k>1                                  │
  ├──────────────────────┼──────────────┼──────────────────────────────────────┤
  │ BFS 次數             │ 2            │ 4                                    │
  │ 動態分配             │ 0            │ 0                                    │
  │ 交集方式             │ bitwise AND  │ bitwise AND                          │
  └──────────────────────┴──────────────┴──────────────────────────────────────┘

  改進倍數：每次呼叫節省 4~6 次 malloc/free；交集計算快 64x


================================================================================
  7. switching randomization
================================================================================

整個 HETA 最耗時的步驟，佔總執行時間 90% 以上。
需執行 n_random=1000 次，每次 Q x m = 100 x m 步。

Python：
  def switching_randomize(G, Q=100):
      Gr    = G.copy()                           # dict deepcopy
      edges = list(Gr.edges())                   # Python list of tuples

      for _ in range(Q * m):
          i, j = random.sample(range(len(edges)), 2)  # Python random + GIL
          a, b = edges[i]
          c, d = edges[j]
          if a == d or c == b: continue
          if Gr.has_edge(a, d) or Gr.has_edge(c, b): continue  # dict 查找
          Gr.remove_edge(a, b)                   # dict 刪除
          Gr.remove_edge(c, d)
          Gr.add_edge(a, d)                      # dict 插入
          Gr.add_edge(c, b)
          edges[i] = (a, d)                      # tuple 重建
          edges[j] = (c, b)

  ┌────────────────────────┬────────────────────────────────────────────────────┐
  │ 圖複製方式             │ Python dict deepcopy                               │
  │ has_edge               │ 兩層 dict 查找，~150 ns/次                         │
  │ 邊交換                 │ dict add/remove，可能 rehash                       │
  │ 隨機數                 │ Python random（GIL，~50 ns/次）                    │
  │ 邊陣列                 │ Python list of tuples（每個 tuple ~72 bytes）      │
  │ 並行                   │ 不可能（GIL）                                      │
  │ 總時間複雜度（1000次） │ O(n_random x Q x m)，Python 常數極大               │
  └────────────────────────┴────────────────────────────────────────────────────┘

C：
  void switching_randomize(Graph *gr, uint64_t seed) {
      Bitset **adj_bs = build_adj_bitset(gr);    // 鄰接矩陣 bitset
      PCG32 rng;
      pcg32_init(&rng, seed, seed ^ 0xCAFEBABEULL);

      for (int step = 0; step < Q * m; step++) {
          int i = pcg32_bound(&rng, m);          // PCG32，無 GIL
          int j = pcg32_bound(&rng, m);
          if (bs_test(adj_bs[a], d)) continue;   // O(1) 位元測試
          if (bs_test(adj_bs[c], b)) continue;
          bs_clear_bit(adj_bs[a], b);            // 位元清除
          bs_set(adj_bs[a], d);                  // 位元設定
          // ...
      }
  }

  // threshold.c：OpenMP 並行處理 1000 個隨機網路
  #pragma omp parallel for schedule(static)
  for (int r = 0; r < n_random; r++) {
      Graph *gr = graph_copy(g);                 // memcpy，極快
      switching_randomize(gr, seed_r);           // 各 thread 獨立執行
  }

  ┌────────────────────────┬────────────────────────────────────────────────────┐
  │ 圖複製方式             │ memcpy（3 個連續陣列）                             │
  │ has_edge               │ Bitset 位元測試，O(1)，~2 ns/次                    │
  │ 邊交換                 │ Bitset 位元運算，O(1)                              │
  │ 隨機數                 │ PCG32（無 GIL，thread-safe，~2 ns/次）             │
  │ 邊陣列                 │ int[] 陣列（每條邊 8 bytes）                       │
  │ 並行                   │ OpenMP，P 核同時計算                               │
  │ 總時間複雜度（1000次） │ O(n_random x Q x m) / P，C 常數極小               │
  └────────────────────────┴────────────────────────────────────────────────────┘

  改進倍數：單核 ~200x；8核 ~1600x


================================================================================
  8. 外部閾值 T_E^k 計算
================================================================================

論文公式 (5)：T_E^k = Mean + 2 x SD
基於 n_random=1000 個隨機網路，是整個演算法計算量最大的步驟。

Python：
  def compute_external_threshold(G, k, n_random=1000):
      rand_ratios = []                        # 先收集所有比例值
      for _ in range(n_random):              # 串列迴圈
          Gr = switching_randomize(G)         # 每次圖複製
          for (u, v) in Gr.edges():
              rand_ratios.append(
                  compute_common_neighbor_ratio(Gr, u, v, k)
              )
      return float(np.mean(rand_ratios) + 2 * np.std(rand_ratios))

  ┌────────────────────────┬────────────────────────────────────────────────────┐
  │ 資料收集方式           │ Python list.append                                 │
  │ 記憶體需求             │ O(n_random x m) float，全存在記憶體                │
  │ mean/std 計算          │ numpy 兩遍掃描                                     │
  │ 並行                   │ 無（GIL）                                          │
  └────────────────────────┴────────────────────────────────────────────────────┘

C（Welford 線上演算法 + OpenMP）：

  Welford 更新公式：
    n    += 1
    delta = x - mean
    mean += delta / n
    M2   += delta * (x - mean)

  #pragma omp parallel
  {
      Welford local_wf;
      welford_init(&local_wf);

      #pragma omp for schedule(static)
      for (int r = 0; r < n_random; r++) {
          Graph *gr = graph_copy(g);
          switching_randomize(gr, seed);
          for (...邊遍歷...) {
              double ratio = compute_cnr(...);
              welford_update(&local_wf, ratio);   // O(1) 空間，一遍完成
          }
          graph_free(gr);
      }

      #pragma omp critical
      welford_merge(&global_wf, &local_wf);       // Chan's parallel formula
  }

  T_E^k = global_wf.mean + 2 x sqrt(global_wf.M2 / global_wf.n)

  ┌────────────────────────┬────────────────────────────────────────────────────┐
  │ 資料收集方式           │ Welford 線上演算法，O(1) 空間                      │
  │ 記憶體需求             │ O(N_thread) 個 double，與 n_random、m 無關         │
  │ mean/std 計算          │ 一遍掃描完成                                       │
  │ 並行                   │ OpenMP，P 核同時計算                               │
  └────────────────────────┴────────────────────────────────────────────────────┘

  改進倍數：記憶體 O(n x m) → O(1)；速度含並行 ~P x 200x


================================================================================
  9. 內部閾值 T_I^k 計算
================================================================================

論文公式 (6)：T_I^k = Mean - SD，基於當前層的候選邊。

Python：
  def compute_internal_threshold(candidate_ratios):
      arr = np.array(candidate_ratios)          # 轉換 numpy array（複製）
      return float(np.mean(arr) - np.std(arr))  # 兩遍掃描

  ┌────────────────────────┬────────────────────────────────────────────────────┐
  │ 輸入                   │ Python list                                        │
  │ 中間轉換               │ np.array（額外複製一次）                           │
  │ 計算                   │ numpy 兩遍掃描                                     │
  └────────────────────────┴────────────────────────────────────────────────────┘

C：
  double compute_internal_threshold(const double *ratios, int n) {
      Welford w;
      welford_init(&w);
      for (int i = 0; i < n; i++)
          welford_update(&w, ratios[i]);         // 一遍掃描，O(1) 空間
      double sd = sqrt(w.M2 / (double)w.n);
      return w.mean - sd;
  }

  ┌────────────────────────┬────────────────────────────────────────────────────┐
  │ 輸入                   │ double[] 陣列（主迴圈即時填入）                    │
  │ 中間轉換               │ 無                                                 │
  │ 計算                   │ Welford 一遍掃描，O(1) 空間                        │
  └────────────────────────┴────────────────────────────────────────────────────┘

  改進倍數：~3x（此步驟本身不是瓶頸）


================================================================================
  10. 主演算法 HETA 流程
================================================================================

Python：
  link_types = {e: None for e in edges}    # dict，key 為 tuple (u,v)
  pass_flag  = {e: True for e in edges}    # dict，key 為 tuple (u,v)

  for (u, v) in edges:
      if G.degree(u) == 1 or G.degree(v) == 1:   # degree = dict 查找
          link_types[(u, v)] = 'silk'
          pass_flag[(u, v)]  = False

  for k in range(1, kmax + 1):
      for (u, v) in edges:
          if not pass_flag[(u, v)]: continue      # tuple hash 查找
          r = compute_common_neighbor_ratio(...)

  ┌────────────────────────┬────────────────────────────────────────────────────┐
  │ 邊狀態存儲             │ Python dict，key = tuple (u,v)                     │
  │ 邊存取                 │ hash(tuple) + dict lookup                          │
  │ degree 查詢            │ len(G._adj[u])，dict 查找                          │
  │ 記憶體/邊              │ ~72 bytes（tuple）+ dict entry                     │
  └────────────────────────┴────────────────────────────────────────────────────┘

C：
  typedef struct {
      int      u, v;
      LinkType type;
      int      layer;
      int      active;    // pass_flag：1=未分類，0=已分類
  } Edge;

  Edge *edges = malloc(m * sizeof(Edge));

  if (g->degree[u] == 1 || g->degree[v] == 1) {    // 直接陣列索引
      edges[i].type   = LINK_SILK;
      edges[i].active = 0;
  }

  for (int i = 0; i < m; i++) {
      if (!edges[i].active) continue;               // 直接陣列存取，O(1)
      double r = compute_cnr(...);
  }

  ┌────────────────────────┬────────────────────────────────────────────────────┐
  │ 邊狀態存儲             │ Edge[] 連續陣列                                    │
  │ 邊存取                 │ 直接陣列索引，O(1)，無 hash                        │
  │ degree 查詢            │ g->degree[u]，int[] 陣列索引                       │
  │ 記憶體/邊              │ 20 bytes（Edge 結構）                              │
  └────────────────────────┴────────────────────────────────────────────────────┘

  改進倍數：記憶體 ~4x；存取速度 ~10x


================================================================================
  11. 平均最短路徑 / kmax
================================================================================

Python：
  avg_spl = nx.average_shortest_path_length(G)
  kmax = max(1, int(avg_spl / 2))
  - NetworkX 底層是 BFS，但透過 Python 層呼叫
  - 每次 BFS 遍歷 dict-of-dict 鄰接表

C：
  double graph_avg_spl(const Graph *g) {
      int *dist  = malloc(n * sizeof(int));
      int *queue = malloc(n * sizeof(int));
      for (int src = 0; src < n; src++) {
          graph_bfs_distances(g, src, dist, queue);
          // 累加距離...
      }
  }
  - n 次 BFS，全部在 CSR 連續記憶體上執行
  - dist 和 queue 陣列預分配一次，n 次 BFS 共用

  ┌────────────────────────┬────────────────┬──────────────────────────────────┐
  │ 指標                   │ Python         │ C                                │
  ├────────────────────────┼────────────────┼──────────────────────────────────┤
  │ 底層 BFS               │ dict-of-dict   │ CSR 連續陣列                     │
  │ 時間複雜度             │ O(n(n+m))      │ O(n(n+m))                        │
  │ 實際速度               │ Python 解釋器  │ 原生 CPU 指令                    │
  └────────────────────────┴────────────────┴──────────────────────────────────┘


================================================================================
  12. 隨機數生成
================================================================================

Python：
  i, j = random.sample(range(len(edges)), 2)
  - Mersenne Twister（MT19937）
  - 每次呼叫：Python 函數呼叫 + GIL 鎖定 + MT 狀態更新
  - 多執行緒下無法並行（GIL）
  - 單次耗時：~50 ns

C（PCG32）：
  // 更新 state
  rng.state = rng.state * 6364136223846793005ULL + rng.inc;
  // 輸出：xorshift + rotation
  uint32_t rot = old >> 59u;
  return (xorshifted >> rot) | (xorshifted << ((-rot) & 31u));

  - PCG32：統計品質優於 MT19937，速度快 2 倍
  - 無 GIL，每個 OpenMP thread 有獨立 PCG32 state
  - Lemire's nearly-divisionless 取模：無偏且無除法
  - 單次耗時：~2 ns

  ┌────────────────────────┬──────────────────────┬──────────────────────────┐
  │ 指標                   │ Python random        │ C PCG32                  │
  ├────────────────────────┼──────────────────────┼──────────────────────────┤
  │ 演算法                 │ MT19937              │ PCG32                    │
  │ 單次耗時               │ ~50 ns（含 GIL）     │ ~2 ns                    │
  │ Thread-safe            │ 否（GIL）            │ 是（獨立 state）         │
  │ 統計品質               │ 良好                 │ 優秀                     │
  └────────────────────────┴──────────────────────┴──────────────────────────┘

  改進倍數：~25x
